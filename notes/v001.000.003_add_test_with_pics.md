### 1. For your own environments

Any other project that reuses that env automatically gets `setuptools==68.2.2`.

### 2. Inside this library

You can’t list `setuptools` as a normal runtime dependency (it’s a build tool), but you can:

- **Pin it in `[build-system]`** so building this project uses a known-good version:

```toml
[build-system]
requires = ["setuptools==68.2.2", "wheel"]
build-backend = "setuptools.build_meta"
```

That won’t control users’ global envs, but it does make builds of *this* package reproducible.

Add a short note in the README “MUSIQ / TensorFlow environment notes”, e.g.:

That’s about as far as you can go without over-constraining other projects.
