`image-analysis score` needs internet because it loads MUSIQ from a remote TensorFlow Hub URL at runtime.

```53:60:image_analysis_lib/musiq.py
def _load_musiq_tf(config: ImageAnalysisConfig = default_config):
    """Lazy-load MUSIQ via TensorFlow Hub."""
    global _MUSIQ_TF
    if _MUSIQ_TF is not None:
        return _MUSIQ_TF
    model = hub.load(config.musiq_model_url)
    _MUSIQ_TF = model.signatures["serving_default"]
    return _MUSIQ_TF
```

```17:19:image_analysis_lib/config.py
# MUSIQ
musiq_model_url: str = "https://tfhub.dev/google/musiq/ava/1"
musiq_default_max_sizes: List[int] = field(default_factory=lambda: [0])
```

So the internet dependency is specifically this:
- `hub.load("https://tfhub.dev/...")` downloads the model on first load (or when cache is missing/corrupt).
- Once cached, scoring can work offline, but only if that cache is present.

How to remove the requirement:

- **Best minimal change:** use a local model path instead of URL.
  - Download MUSIQ once to a local directory (SavedModel format).
  - Set `musiq_model_url` to that local path (same `hub.load(...)` call can load local handles).
  - Result: no network needed during scoring.
- **No code change, operational workaround:** prewarm and persist TF Hub cache.
  - On a machine with internet, run one score once.
  - Keep `TFHUB_CACHE_DIR` on durable storage and copy it to offline machines.
  - Result: offline works as long as cache is intact.
- **More explicit offline mode (code change):**
  - Add config like `musiq_model_path` and prefer it over `musiq_model_url`.
  - Fail fast with a clear error if offline and local model not found.
  - Optionally remove URL default entirely so internet is never attempted.

If you want, I can map out the exact patch to add `--musiq-model-path` to the CLI so `image-analysis score` is guaranteed offline-capable.
